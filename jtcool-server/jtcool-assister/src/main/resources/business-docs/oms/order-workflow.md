# 订单审批流程

## 概述

订单从创建到完成需要经过7个状态，每个状态由不同的角色负责确认和推进。

## 状态流转图

```
已下单 → 销售确认 → 订单审核 → 仓库确认 → 登记出库 → 确认发货 → 客户签收
```

## 详细流程

### 1. 已下单 (PLACED)
- **说明**: 客户下单后的初始状态
- **负责角色**: 系统自动创建
- **下一步**: 销售确认
- **操作**: 销售人员需要确认订单信息（客户、商品、数量、价格）

### 2. 销售确认 (SALES_CONFIRMED)
- **说明**: 销售人员确认订单信息无误
- **负责角色**: 销售人员 (ROLE_SALES)
- **下一步**: 订单审核
- **操作**: 审核人员检查订单合规性、库存可用性

### 3. 订单审核 (ORDER_REVIEWED)
- **说明**: 订单通过审核，可以进入仓库处理
- **负责角色**: 审核人员 (ROLE_AUDITOR)
- **下一步**: 仓库确认
- **操作**: 仓库人员确认可以备货出库

### 4. 仓库确认 (WAREHOUSE_CONFIRMED)
- **说明**: 仓库确认可以出库，系统自动创建出库单
- **负责角色**: 仓库人员 (ROLE_WAREHOUSE)
- **下一步**: 登记出库
- **触发动作**: 自动创建WMS出库单
- **操作**: 仓库人员完成拣货、打包后登记出库

### 5. 登记出库 (OUTBOUND_REGISTERED)
- **说明**: 货物已出库，等待发货
- **负责角色**: 仓库人员 (ROLE_WAREHOUSE)
- **下一步**: 确认发货
- **操作**: 物流人员确认发货并录入物流单号

### 6. 确认发货 (SHIPMENT_CONFIRMED)
- **说明**: 货物已发出，等待客户签收
- **负责角色**: 物流人员 (ROLE_LOGISTICS)
- **下一步**: 客户签收
- **操作**: 客户签收后由系统或客服确认

### 7. 客户签收 (CUSTOMER_RECEIVED)
- **说明**: 订单完成
- **负责角色**: 客户服务 (ROLE_CUSTOMER_SERVICE)
- **下一步**: 无（终态）

## 业务规则

1. **状态只能单向推进**，不能回退
2. **每个状态转换需要相应角色权限**
3. **仓库确认后自动创建出库单**，无需手动操作
4. **订单取消**：只有在"已下单"和"销售确认"状态可以取消
5. **异常处理**：如果发货后客户拒收，需要走退货流程（另外的工作流）

## 常见问题

**Q: 如何查询订单当前状态？**
A: 使用订单ID查询，系统会返回当前状态和操作历史。

**Q: 订单卡在某个状态怎么办？**
A: 联系对应角色的负责人处理，或联系管理员查看审批记录。

**Q: 可以跳过某个状态吗？**
A: 不可以，必须按顺序推进以保证业务合规性。
