# 性能约束

## 已完成的性能优化

### Phase 1: 连接池优化（已完成）
- 数据库连接池已优化，支持 60 并发用户
- 新功能开发无需再调整连接池配置

### Phase 2: 异步导出基础设施（已完成）
- 异步导出框架已就绪
- **所有新的导出功能必须使用异步导出**
- 不要再使用同步导出方式

### Phase 3: OMS/WMS 导出优化（已完成）
- OMS 订单导出已优化
- WMS 库存导出已优化

## 性能开发约束

### 数据库查询
- **必须为查询字段添加索引**
- 避免 `SELECT *`，只查询需要的字段
- 大数据量查询使用分页
- 统计查询必须添加时间范围限制

### 导出功能
- **强制要求**：所有导出功能使用异步导出
- 导出数据量超过 1000 条时，必须分批处理
- 导出任务状态可通过接口查询

### 并发处理
- 系统已支持 60 并发用户
- 高并发接口需要考虑缓存策略
- 避免在事务中执行耗时操作

## 性能监控

### 关键指标
- 接口响应时间应 < 500ms
- 数据库查询时间应 < 200ms
- 导出任务应在后台异步执行

### 优化建议
- 使用 Redis 缓存热点数据
- 定期清理历史数据
- 监控慢查询日志
